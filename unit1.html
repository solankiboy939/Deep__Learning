Unit 1: Foundations of ML & TensorFlow

Master the core concepts of machine learning, tensors, computation graphs, and optimizers — the building blocks of modern AI.

1. What is Machine Learning?

Machine Learning is about creating algorithms that learn patterns from data instead of following explicitly programmed rules.

Example: Instead of coding "if red AND round → apple", you show the computer thousands of labeled images of apples and oranges, and it learns the distinguishing features itself.

Key Applications
  • Prediction: Forecasting stock prices or weather
  • Classification: Spam detection, medical diagnosis
  • Clustering: Customer segmentation
  • Recommendation: Netflix, Amazon suggestions
2. TensorFlow: The ML Engine

TensorFlow is Google's open-source library for building and training machine learning models, especially deep neural networks.

The 3-Step Lifecycle
  1. Build: Design your model architecture
  2. Train: Feed data and let the model learn
  3. Deploy: Integrate into apps for real predictions

Tensor Properties
  • Rank: Number of dimensions (scalar=0, vector=1, matrix=2)
  • Shape: Size of each dimension (e.g., [32, 28, 28, 3] for image batch)
  • dtype: Data type (tf.float32, tf.int32, etc.)
4. Computation Graph: The Engine

TensorFlow represents calculations as a graph where:

  • Nodes = Operations (add, multiply, relu, etc.)
  • Edges = Tensors (data flowing between operations)

This structure enables automatic differentiation (backpropagation) — the magic that allows neural networks to learn!

5. Variables vs Constants
# Constants (fixed values)
constant_tensor = tf.constant([[1.0, 2.0], [3.0, 4.0]])

# Variables (learnable parameters)
weights = tf.Variable(tf.random.normal([784, 10]))
bias = tf.Variable(tf.zeros([10]))

# Basic operations
result = tf.add(a, b)
output = tf.matmul(input, weights) + bias

Variables hold the weights and biases that get updated during training. Constants are fixed values that don't change.

7. Deep Learning & Neural Networks

Deep Learning uses neural networks with many layers to automatically learn hierarchical features from data.

Early layers learn simple features, later layers learn complex concepts.

8. Vanishing Gradient Problem

In deep networks, gradients can become extremely small as they backpropagate through many layers, causing early layers to stop learning.

Solutions
  • ReLU Activation: f(x) = max(0, x) — prevents gradient from vanishing for positive values
  • Batch Normalization: Normalizes layer inputs to stabilize training
  • Residual Connections: Skip connections in ResNet architectures
  • Better Initialization: He or Xavier initialization
# Using ReLU activation
model.add(tf.keras.layers.Dense(128, activation='relu'))

# Adding Batch Normalization
model.add(tf.keras.layers.BatchNormalization())
